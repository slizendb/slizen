# Support

Slizen is a developer preview without a production support SLA.

- Use the [bug report form](https://github.com/slizendb/slizen/issues/new?template=bug.yml) for reproducible incorrect behavior.
- Use the [feature proposal form](https://github.com/slizendb/slizen/issues/new?template=feature.yml) for a measured workflow or reliability need.
- Use the [design-partner form](https://github.com/slizendb/slizen/issues/new?template=design-partner.yml) to evaluate a real hot-key workload with the maintainers.
- Use [private vulnerability reporting](https://github.com/slizendb/slizen/security/advisories/new) for suspected security issues. Never put exploit details, credentials, values, or sensitive keys in a public issue.

Include the Slizen version and full commit, Redis or Valkey version, deployment shape, redacted configuration, minimal reproduction, and sanitized evidence. Best-effort community support does not imply a response-time commitment.
